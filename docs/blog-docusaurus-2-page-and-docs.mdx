## Page 

`Page`란 단일 페이지를 의미한다. 자식이 있는 페이지를 만들려면 `Docs`로 만들어야 한다.

각 페이지는 리액트 컴포넌트일 수도 있다.

블로그나 문서가 아닌 페이지가 필요한 경우에만 `Page`를 쓸만 해서, Docusaurus를 쓰는 경우는 쓸 일이 잘 없을 것 같긴 하다.

### 만드는 법

https://docusaurus.io/docs/creating-pages

#### 리액트 컴포넌트로 페이지 만들기

페이지는 `src/pages` 디렉토리 밑에 있는 모든 `mdx?`와 `(j|t)sx?` 파일이 대상이 된다.

따라서 컴포넌트는 `pages` 밑에 놓지 않아야 한다.

여기서 페이지를 정의하면 다른 곳에서 자동으로 링크되진 않는다(`docs`, `blog` 등).

각 페이지는 완전히 비어있는데, `<Layout>` 컴포넌트를 직접 렌더링해야 한다.

#### md(x)로 페이지 만들기

아래처럼 Frontmatter를 작성한다. 대충 `id`, `title`, `description` 정도만 작성하면 되는 것 같다.

```md
---
title: my hello page title
description: my hello page description
hide_table_of_contents: true
---

# Hello

How are you?
```

## Docs

`Docs`도 그냥 페이지이다.

### Doc 별 부가 설정

읽어보니 굳이 안 쓸 것 같은 내용이다.

오히려 기본 값이 좋아서 잘 정리해주는 방식 그대로 쓸 것 같다.

- https://docusaurus.io/docs/create-doc
- `tag`를 직접 설정할 수 있다.
- 폴더 구조대로 Sidebar 구성이 된다.
- URL을 `slug` 옵션으로 바꿀 수 있다.

### Sidebar 설정

굳이 설정할 필요는 없는 거 같다.

autogenerated가 문제가 되는 경우에 봐야할 것 같다.

할 게 테마 설정 정도인데, 아래 페이지 보면 사이드바 접기 UI 사용 여부가 있다.

https://docusaurus.io/docs/sidebar#theme-configuration

기본 값은 아니지만 hideable 안 하면 타블렛 크기에서 안 사라지나 보다.

또 다른 Doc 열었을 때 기존 Doc이 안 닫힌다거나 등등

## blog

### 블로그 Sidebar 설정

블로그도 Sidebar가 있다. 최근 글을 보여주는 건데 그렇게 쓸모가 있지는 않다.

https://docusaurus.io/docs/blog#blog-sidebar

### 요약글 분량 설정

https://docusaurus.io/docs/blog#blog-list

되게 재밌는 게 블로그는 기본 뷰가 ListView 이다. 이 때 ListItem의 보여줄 내용 일부를 지정할 수 있는데, 이를 `<!-- truncate -->`로 쓴다.

### author 설정

블로그는 재미있는 게 저자를 직접 설정할 수 있다. 애초에 설정 안 하는 게 더 근본 있어보이긴 한데..

아래처럼 frontmatter에 지정할 수 있다.

지정하면 꽤 예쁜 프로필이 나오기 때문에 굳이 커스텀하거나 안 쓸 이유는 없는 거 같다.

```mdx
---
authors:
  name: Joel Marcey
  title: Co-creator of Docusaurus 1
  url: https://github.com/JoelMarcey
  image_url: https://github.com/JoelMarcey.png
  email: jimarcey@gmail.com
---
```

### Docs Category URL 변경하기

https://docusaurus.io/docs/sidebar/autogenerated#category-item-metadata

`autogenerated`면 변경 못 하는 거 같다...

그렇다고 모든 글에 대해 Sidebar에도 같이 정의하기에는 글을 쓸 때 큰 불편함이 생기게 된다.

일단은 `autogenerated`로 쓰고 이후에 Docusaurus 소스를 파악해보면서 커스텀해봐야겠다.
